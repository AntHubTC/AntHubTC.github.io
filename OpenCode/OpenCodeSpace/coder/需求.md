### uni-app 本地记账 App

> 你是一名 **精通 uni-app（Vue3 + Composition API）与 HBuilderX App 开发的高级前端工程师**。
>
> 请你从 **工程设计 + 数据库设计 + 页面实现** 的角度，帮我 **完整设计并实现一个「本地记账 App」**，要求如下：
>
> ---
>
> ## 一、技术与运行环境
>
> * 使用 **uni-app（Vue3 + Composition API）**
> * 运行平台：**App（Android / iOS）**
> * **不使用任何后端服务**
> * 所有数据存储在 **plus.sqlite 本地数据库**
> * 使用 `plus.sqlite.open / executeSql / selectSql`
> * 项目可直接在 **HBuilderX 真机运行**
>
> ---
>
> ## 二、数据库设计（重点）
>
> 请你：
>
> * 设计合理的 **SQLite 数据库结构**
> * 给出完整的 `CREATE TABLE` SQL
> * 考虑后续扩展（统计、同步）
>
> 至少包含账单表（示例字段）：
>
> * id INTEGER PRIMARY KEY AUTOINCREMENT
> * type TEXT（income / expense）
> * amount REAL
> * category TEXT
> * note TEXT
> * date TEXT（YYYY-MM-DD）
> * create_time INTEGER
>
> 请说明：
>
> * 数据库初始化逻辑
> * 首次启动建表方案
> * 数据库打开 / 关闭时机
>
> ---
>
> ## 三、功能需求
>
> ### 1️⃣ 记账功能
>
> * 新增收入 / 支出
> * 表单字段：金额、分类、日期、备注
> * 基础校验（金额、日期必填）
> * 提交后写入 SQLite
>
> ### 2️⃣ 账单列表
>
> * 从 SQLite 读取账单
> * 按日期倒序排列
> * 支持删除账单
> * 支持按日期分组展示
>
> ### 3️⃣ 统计功能（含图表）
>
> #### 数字统计
>
> * 本月收入
> * 本月支出
> * 本月结余
>
> #### 图表统计（重点）
>
> * **分类支出饼图**
>
>   * 本月各分类支出占比
>   * 使用 SQL：`SUM + GROUP BY category`
>   * 使用 **canvas 原生绘制**
> * **最近 7 天支出趋势图**
>
>   * 折线图或柱状图均可
>   * 无数据日期补 0
>   * 使用 canvas 原生绘制
>
> ❗ 不使用任何第三方图表库（echarts / uCharts 等）
>
> ---
>
> ## 四、页面与结构设计
>
> 请给出：
>
> * 页面路由结构
> * 页面职责说明
>
> 页面至少包含：
>
> * 账单列表页（首页）
> * 新增记账页
> * 统计页（含图表）
>
> ---
>
> ## 五、工程与代码要求
>
> * 封装统一的 **SQLite 工具模块**（如 `utils/db.js`）
> * 所有 SQL 操作集中管理
> * 使用 `async / await`
> * 关键逻辑添加 **中文注释**
> * 代码风格清晰，适合长期维护
>
